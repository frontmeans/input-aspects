Class InSubmit<Value>

Input submit aspect.

Allows to submit input data and reports submit status.

Implements an EventKeeper interface by sending submit status flags to registered receivers.

Static Submitter

Submitter<Value, Result>: function
  • Defined in submit.aspect.ts:180

Input submitter function interface.

A submitter is passed to InSubmit.submit() method to perform the actual submit.

A submit failure (a returned promise rejection) is reported as validation messages. This can be either a [InSubmitError], or arbitrary error. Previously reported submit messages are replaced by the reported ones, and cleared on a new submit.

param

Input data to submit.

param

Input control the submit is performed for.

returns

Submit result promise.

Abstract submit

  • submit<Result>(submitter: Submitter<Value, Result>): Promise<Result>
    • Defined in submit.aspect.ts:117

    Attempts to submit the input data.

    1. Sets submitted status flag.
      1. Rejects to submit if busy status flag is set, or if ready status flag is not set with [InSubmitRejectedError]. Note that the latter mey be reset after the step 1. So this check is performed asynchronously.
      1. Otherwise, sets the busy status flag.
    4. Clears submit failure messages.
    5. Calls submitter function.
    6. Waits for the promise returned by submitter to resolve.
      1. Resolves the result promise if submit were successful.
      1. Otherwise, reports submit failure messages.
    9. Rejects the result promise.
